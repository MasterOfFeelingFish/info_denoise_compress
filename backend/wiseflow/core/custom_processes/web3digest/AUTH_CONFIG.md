# 用户鉴权配置指南

## 📋 概述

系统已实现完整的用户鉴权功能，包括：
- 用户角色管理（普通用户、管理员、超级管理员）
- 用户状态管理（激活、未激活、封禁、暂停）
- 白名单/黑名单机制
- 访问权限控制

## 🔑 配置管理员

### 方式 1：环境变量（推荐）

在 WiseFlow 项目根目录的 `.env` 文件中配置：

```bash
# 管理员 Telegram ID（逗号分隔）
ADMIN_TELEGRAM_IDS=123456789,987654321

# 是否启用白名单模式（默认 false，所有用户可访问）
ENABLE_WHITELIST=false
```

### 方式 2：通过代码设置

管理员也可以通过 `auth.json` 文件配置，系统会自动创建该文件。

## 👥 用户角色

### 角色类型

1. **USER (普通用户)**
   - 默认角色
   - 可以使用所有基础功能

2. **ADMIN (管理员)**
   - 可以访问管理员面板
   - 可以管理用户、白名单、黑名单
   - 可以查看系统统计

3. **SUPER_ADMIN (超级管理员)**
   - 拥有所有管理员权限
   - 可以通过环境变量配置

## 📊 用户状态

### 状态类型

1. **ACTIVE (激活)**
   - 默认状态
   - 用户可以正常使用服务

2. **INACTIVE (未激活)**
   - 用户暂时无法使用服务
   - 可以重新激活

3. **BANNED (封禁)**
   - 用户被永久封禁
   - 无法访问服务

4. **SUSPENDED (暂停)**
   - 用户被临时暂停
   - 可以解除暂停

## 🔐 访问控制

### 白名单模式

如果启用白名单模式（`ENABLE_WHITELIST=true`），只有白名单中的用户才能访问服务。

**使用场景**：
- 内测阶段
- 邀请制服务
- 限制访问范围

### 黑名单

黑名单中的用户无法访问服务，无论其他设置如何。

**使用场景**：
- 封禁恶意用户
- 临时禁止访问

## 🛠️ 管理员功能

### 管理员命令

在 Telegram Bot 中发送 `/admin` 命令，可以访问管理员面板：

- **用户管理**: 查看所有用户及其状态
- **白名单管理**: 添加/移除白名单用户
- **黑名单管理**: 添加/移除黑名单用户
- **系统统计**: 查看用户统计和系统状态

### 管理员操作

管理员可以通过管理员面板进行以下操作：

1. **设置用户角色**
   - 将普通用户提升为管理员
   - 撤销管理员权限

2. **设置用户状态**
   - 激活/暂停/封禁用户

3. **管理白名单**
   - 添加用户到白名单
   - 从白名单移除用户

4. **管理黑名单**
   - 添加用户到黑名单
   - 从黑名单移除用户

## 📝 配置示例

### 完整配置

```bash
# .env 文件

# 管理员配置
ADMIN_TELEGRAM_IDS=123456789,987654321

# 白名单模式（可选）
ENABLE_WHITELIST=false
```

### 获取 Telegram ID

1. 在 Telegram 中搜索 `@userinfobot`
2. 发送任意消息
3. Bot 会返回您的 Telegram ID

## 🔍 鉴权流程

### 用户访问流程

```
用户发送命令
    ↓
检查黑名单
    ↓ (如果在黑名单中)
拒绝访问 ❌
    ↓ (如果不在黑名单中)
检查白名单模式
    ↓ (如果启用白名单)
检查是否在白名单中
    ↓ (如果不在白名单中)
拒绝访问 ❌
    ↓ (如果在白名单中或未启用白名单)
检查用户状态
    ↓ (如果状态为 BANNED 或 SUSPENDED)
拒绝访问 ❌
    ↓ (如果状态为 ACTIVE)
允许访问 ✅
```

### 管理员权限检查

```
用户请求管理员功能
    ↓
检查用户角色
    ↓ (如果不是 ADMIN 或 SUPER_ADMIN)
拒绝访问 ❌
    ↓ (如果是管理员)
允许访问 ✅
```

## 📊 数据存储

鉴权数据存储在 `{DATA_DIR}/auth.json` 文件中：

```json
{
  "users": {
    "123456789": {
      "role": "admin",
      "status": "active"
    }
  },
  "whitelist": ["123456789"],
  "blacklist": ["987654321"],
  "admin_ids": [123456789]
}
```

## 🛡️ 安全建议

1. **保护管理员 ID**
   - 不要将管理员 ID 提交到代码仓库
   - 使用环境变量配置

2. **定期审查**
   - 定期检查管理员列表
   - 审查白名单/黑名单

3. **日志监控**
   - 监控权限拒绝日志
   - 关注异常访问尝试

4. **最小权限原则**
   - 只授予必要的管理员权限
   - 定期撤销不再需要的权限

## 🔧 故障排查

### 问题 1: 无法访问服务

**症状**: 提示"您没有访问权限"

**解决**:
1. 检查是否在黑名单中
2. 如果启用白名单，检查是否在白名单中
3. 检查用户状态是否为 ACTIVE

### 问题 2: 无法使用管理员功能

**症状**: 提示"您没有管理员权限"

**解决**:
1. 检查 Telegram ID 是否在 `ADMIN_TELEGRAM_IDS` 中
2. 检查 `auth.json` 中的管理员列表
3. 确认用户角色为 ADMIN 或 SUPER_ADMIN

### 问题 3: 白名单不生效

**症状**: 白名单用户仍无法访问

**解决**:
1. 检查 `ENABLE_WHITELIST` 是否为 `true`
2. 确认用户 ID 在白名单中
3. 检查用户是否同时在黑名单中（黑名单优先级更高）
